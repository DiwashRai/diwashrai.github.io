Creating a Hugo Blog with your own layout

A Hugo blog is a great way to learn more about HTML and CSS before jumping into a CSS framework. You can even host the site for free using GitHub pages and come out of it with a blog/site that you can be proud of. This is what I wanted to do when I created my blog.

However, after reading some guides and watching some videos, I had no idea where to start.

The Home page? The Layout? The navigation bar? Styling the HTML elements?

In reality, it might not matter which area you tackle first. What does matter is that this confusion can delay or stop this endeavour.

What I’ve presented here is the incremental steps that I took when creating my own blog. What I won’t be covering is the specific techinical implementation details for each step. I feel this is handled already else and far better than I can.

  1. Create blank hugo site folder structure
  2. Put in some dummy content
  3. Create a skeleton of the site
  4. Choose a layout for your site
  5. CSS styling for the navigation bar
  6. CSS styling for the list elements
  7. Create markdown guide/cheat sheet as a first practice post
  8. Start posting

The main advantage to this approach is actually that after each step you can see satisfying progress beng made which is very motivating.

Although I won’t be going into the nitty gritty implementation details here are a few links that could help you out when following these steps.

  • Official quickstart guide
  • Hugo introductory YouTube series
  • Making a hugo blog from scratch walkthrough

1. Create blank hugo site folder structure

The first step is very simple and is just to ensure the same blank starting point. Create your hugo website project folder with the following command.

hugo new site [path] [flags]

You should be left with a folder structure similar to the one shown below.

2. Put in some dummy content

The next step is to create some dummy content that you can use to show the HTML or CSS changes you will be making.

As an example I have created four sections for each season and a William Shakespeare poem that lives in each of those sections.

hugo new autumn/Sonnet-73.md

3. Create a skeleton of the site

The point of this next step is get to a point where there is zero CSS styling, however, you can still navigate the site and the dummy content you have created so far.

4. Choose a layout for your site

This next step is where you actually have to decided on a layout for your site and implement it.

Will it have the navigation bar at the top or on one side?
Where will the posts be listed and where will the content be shown?

This may be a good opportunity to brush up on CSS grid and flexbox to help you do this.

5. CSS styling for the navigation bar

Now we finally get to dive into some CSS. This part really needs no elaboration. Write the CSS for your site’s navigation bar.

6. CSS styling for the list elements

The next step is to do the styling for how each post element will look when it is listed on your site.

By now you should notice that your site is slowly starting to take form.

You could start posting things on your blog and it would technically be ‘functional’.

However, as the contents of the actual post are still unstyled, we have one more step.

7. Create markdown guide/cheat sheet as a first practice post

Finally you need to do the CSS for all the HTML elements that could be involved in a blog post.

A good way to do this is to copy one of those markdown cheatsheet/guide pages and ensure that the markdown is styled on your site too.

8. Start posting!

Your site should now be in a position where you can start posting and you should start as soon as you can.

Ofcourse there’s still more you can add to it. A few of these things could be to make it responsive, pagination, dark mode etc.
